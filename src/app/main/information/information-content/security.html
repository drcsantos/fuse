Authentication

Apila thinks about user authentication the same way a Bank would think about it.

  • 100% HTTPS
  • Passwords stored with hash and salt encryption
  • Destroy user session after inactivity
  • Destroy user session after close of tab and browser
  • Destroy user session on password resets
